Node.js NPM


  • Node.js 原有內建核心功能不夠用，所以有其他 Node.js 的開發者開發了其它功能，可供其他開發者下載。 NPM
  • 管理模組的套件 (package.json)
  • 下載 node 就內建下載 npm

npm init >> package.json

  • 在我們下載 npm 的模組(功能)之前，要先有一個管理者 package.json，可以記錄我們安裝了什麼模組。
  • HOW? >> 在終端機輸入 npm init ，輸入相關資訊。 OK 後會發現我們的資料夾內出現 package.json

正式來安裝(下載) npm 的模組

  • 這邊以 npm 的 express 來作範例:
    express

  • 安裝後，在 package.json 會多了 dependencies，並記錄該才下載的 express
  • 版本號說明如下:
    • ^4.17.1
      • 4:主要版本號
      • 17:次要版本號(小功能)
      • 1:bug修正
      • ^符號:安裝 4.x.x 自動更新後面版本(除了主要版本號)
      • ~符號:只安裝最後 bug 的版本號 (怕更新模組會影響到程式)
      • 只顯示版本號:永遠都是這個版本號
      • latest: 永遠都是最新的版本(很少用)
  • 在根目錄資料夾內會多一個 node_modules 的資料夾，裡面除了這次下載的 express，也有其他 express 相依的檔案。

很大一包的 node_modules

  • 不會進版控(忽略)。
  • 只要有 package.json 紀錄，其他開發者只需輸入 npm install 即可把我們有下載的模組都下載起來。

下載>> 正式、測試、全域

如圖所示，此為 –save-dev 在 package.json的狀況

  • 若是 Node.js 的新手，還沒有開發很久，不曉得自己在測試環境會用到什麼模組，可先用 –save
  • 模組安裝在全域環境並不會記錄到 package.json，雖然在此台電腦的所有專案都可以用到； 但如果這包檔案是需跟同事配合(協同)狀況的話，就不建議下載到全域，因為這時同事 npm install 實際可能會缺模組無法使用。(畢竟有的模組可能是運行在我們電腦的全域內)

nodemon 不需每次都重整 (類似 vscode 的 golive)

內容更動時，就不需 ctrl+c 取消再重啟。

npm 補充指令

  • npm list ：顯示安裝的 NPM 列表
  • npm uninstall [模組名稱] ：刪除專案裡的 NPM
