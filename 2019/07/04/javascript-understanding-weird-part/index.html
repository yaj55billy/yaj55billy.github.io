語法解析器: 我們寫的 Js 程式，中間有一個轉譯器幫我們把 Js編譯成電腦懂得語言。

詞彙環境: 程式碼的實際位置，它被寫在哪裡? 它的周圍環境是什麼? (記憶體)

執行環境: 管理正在執行的程式 (ing)，除了我們寫得程式碼，執行環境還有做其他事情。

  • 執行環境(創造):
    • Js在執行環境的一開始，就為我們建立了 window全域物件、特殊變數 this
    • this變數指向 window物件
    • 外部環境
    • Hoist 變數、函數
  • 執行環境(執行):
    • 逐行執行我們寫的程式碼
    • 單執行序、同步 : 一次只做一件事情，而且是按照順序的做

Hoist

  • 當語法解析器執行，並開始轉化我們寫的程式碼時，這時它就會將我們的變數、函式寫在記憶體內，這個稱為 Hoisting，而在 Hoisting中，變數的處理方式較為奇怪，它需要等到真正執行時，才會知道值是什麼。 (在那之前會先丟一個 undefined)
    • 情境: 一個包裹寄到了一間商店，它在這間商店佔有了空間(記憶體)，但是在我們打開(執行)之前，我們不知道它是什麼(尚未定義)，當我們打開後才會知道這個包裹是餅乾、或者是飲料、又或者是其它東西。
  • undefined >> 一個 Js 的特殊值，當記憶體有這個變數，就會先被指派這個詞。

函數呼叫與執行堆 (運行的流程)

  • 全域環境被建立 >> a的執行環境 >> b的執行環境 (環境會不斷堆疊上去) >> b的執行環境結束 >> a的執行環境(尚未執行的) >> a的執行環境結束 >> var d
  • 執行環境會在流程中，不斷堆疊上去 >> 執行堆

scope chain 範圍鏈

  • (程式的實際位置，以及變數參照的外部環境是什麼，會依照實際範圍位置的”範圍鏈”參照外部環境)

  • 上方的範例，首先我們先執行全域環境 >> a 的執行環境 >> b的執行環境 (這時 b 的 furnitureTv 會是什麼值?)

  • 我們用一個情境來看: 每個 function 先把它看作是一個房子(不同的執行環境)，我們在各自的家裡擁有的電視(變數 furnitureTv )是不同的，電視(變數)只存在這個環境，如果說當我們要看電視(尋找變數)，發現沒有電視時，這時我們並不會到別人房子( function b 不會去 function a)找，而我們會去外面(外部環境)去尋找跟購買。
  • 情境: 這個狀況就像 a 住家，裡面有一間 b 房間(b 的執行環境是在 a 的內部)，如果我們在 b 的房間時想找電視看，我們並不會跑到外面去找電視，所以第一個選擇會是在自己住家買的 LG 電視 (b 會先參照 a 的環境 )。

非同步的概念

  • 如前面提到的，執行環境執行時，Js為同步、事情為一件一件的做。

  • JS 在瀏覽器執行時依賴引擎來驅動，而瀏覽器不是只有JavaScript引擎的存在，還有其他引擎在處理瀏覽器的其他狀態、程式。例如處理渲染的引擎、Http，此時 JS 引擎是可以跟其他引擎做非同步的互動，但 JS 引擎本身是同步的，那它會怎麼處理非同步的操作呢?

  • 非同步要操作的會被 JS引擎放置在事件佇列Event Queue(例如滑鼠點擊事件)，等到執行環境的程式都執行完成後，JS 才會開始跑事件佇列，即使我們在網頁一打開就馬上點擊按鈕，它還是會等到執行環境的部分都完成，最後才會來跑滑鼠點擊的事件。

型別與運算子

  • JS跟其他語言上算是很不同的，尤其是變數的資料、型別部分
  • JS是動態型別語言，我們在宣告變數時，不需要特意宣告型別，在執行時就會自動判別型別。

型別:

  • undefined: 表示未定義，是 JS 給所有變數的初始值，開發者最好不要賦值undefined，最後可能會搞不清楚，這是 JS 給的? 還是我們自己定義的?
  • null: 表示不存在，開發者在宣告變數並要先表示這個變數沒有值時，可以賦值null。
  • boolean: true或false
  • number: 表示數字，JS的數字型別只有number，不像其他程式數字還有分整數與其他特定數值型態，另外JS的number是浮點數，表示(實際上)有小數點跟在後面。
  • string: 表示字串，由字符組成，可以用單引號或雙引號來表示。
  • symbol: 表示符號，是ES6新增的型別，可以賦予變數獨特性、獨一性。

運算子:

  • 是一個特別的函數，會將兩個參數做運算，並 return 出一個結果
  • 算術運算子、賦值運算子、比較運算子、邏輯運算子 (加、減、乘、除)
  • 可以把運算子想像成是函式的一種，這些符號會將前後兩個參數傳入對應的JS內建的函式中，進行個別的運算並回傳。
  • 運算子的兩個特性: 優先性與相依性
    • 優先性: 表示哪一個運算子被優先運算，當同一行程式有不只一個運算子時，具高優先性的運算子會先計算，然後依序算到排序等級低的運算子。 (ex: 先乘除後加減、括號的運算子優先計算…)
    • 相依性: 表示運算子被計算的順序，若是左到右計算的運算子就稱為左相依性，右到左則稱為右相依性。(如果運算子有相同優先性，這時就會看相依性)

運算子優先性表格網址

函式預設值

  • 滿常看到人家用，但自己很少這樣寫(可能是 JS 還寫得不夠多)

  • 我們沒有帶值進去，JS 在呼叫函式時也沒報錯，因為傳入的值在呼叫階段被設定進記憶體，但我們什麼都沒有傳，所以 JS 把這個空的東西，在記憶體被設定成 undefined 並傳入。當 undefined 與字串相加時被轉型成字串’undefined’，自然印出’Hello undefined’囉！

  • 也就是說用這種方法，開發者可以在不影響結果的狀況，讓 JS 函式內有有預設值，不管有沒有帶值進去，都不會被 undefined 影響呼叫函式的結果。
  • 特別的 || 運算子，他會在兩邊的值內，輸出 true 的值。

物件與函數

  • 物件 >> 名稱與值的組合 >> 值還可以是其他名稱/值的組合

    • (在其它語言上，物件可能還有其他定義)

  • 簡單來說，物件內可以有純值、另一個物件、函式，而原本的物件能透過記憶體的位置去取用到這些值、物件、函式(方法)。

  • 當函式在物件裡，稱為方法Methods。
  • 點運算子 (body.arm) >> 在物件 body 內去找 arm 的屬性 (在身體找手臂)
    • 在物件去取用成員(屬性)
  • 當語法解析器，看到一個大括號時，如果周圍沒有看到 if 或者是函式時，他就會認定這個是物件。
  • 這種宣告目前有想到在 jQuery跟一些外掛滿常看到的

JSON與物件實體

  • JSON，是受到 JavaScript 物件實體語法啟發的傳輸格式，比起使用XML傳輸資料，JSON格式在檔案大小上更為輕量，也是現在主流的傳輸格式。(接後端 API 有時會用)
  • 如果在物件包了引號，這是可允許的； 但 JSON 是一定要在屬性的地方有引號。 ( JSON 規則比較嚴格)
  • 雖然JSON是受到JavaScript啟發，但JSON和JavaScript是不一樣的東西。
  • JSON.stringify() 將JS物件轉為JSON字串

  • JSON.parse(); 將JSON字串轉為JS物件

  • 函式為一個特殊形態的物件，它有所有物件的特色，還有一些其他屬性。

函式陳述句

傳值 by value 與傳址 by reference (關於變數)

call by value

  • 當我們創造變數並給值時，變數會指向值在電腦記憶體中的位置，若我們以這個值為參照，指定另一個變數指向這個值時，電腦會在記憶體中新增(複製)一個新值，讓後來的這個變數指向新的值。
    • 在JavaScript裡，布林值、字串、數值、null、undefined都是call by value。

call by reference

  • 當我們創造變數並給值(物件)時，變數會指向物件在電腦記憶體中的位置，若我們以這個物件為參照，指定另一個變數指向這物件，這個變數就會指向電腦記憶體中同樣的物件，不會有新的物件在記憶體中被創造出來。
    • 在JavaScript裡，物件、陣列、函式都是call by reference。
    • 當我們給變數一個物件，其實賦予變數指向這個物件在電腦記憶體的位址，而d = c，變數d也指向同一個物件，同一個該物件在記憶體的位址。

物件、函式與「this」

  • JavaScript 在建立執行環境時，不論是全域、區域執行環境，在創造時會一併建立一個變數 this。而 this 會指向呼叫函式的執行環境，更進一步的說，this 會指向函式目前所在物件。
  • 這時 this 變數就會指向 c 物件。

陣列

  • 表示許多東西的集合
  • JS 的陣列為動態型別，不需要特別宣告我是數值型別或者其他型別，在陣列內可放不同型別的東西。
  • 從 0 開始算

arguments (參數) and spread與分號

  • 執行環境開始時，JS 引擎幫我們設定了環境(像是變數、外部參照、this的指向等等)，他還幫我們建立了一個東西 >> arguments
    • 參數包含了所有傳入到所呼叫的函數之值

立即呼叫的函式表示式(IIFE)

  • Immediately Invoked Functions Expressions
  • 何時會出錯? >> IIFE
  • 函式表示式
  • 這樣就不會報錯了，這也是IIFE，也是最常在JS框架、套件看到的寫法。
  • 使用立即函式的好處，除了可以立即執行程式碼，省略多餘的呼叫，還可以用來避免汙染全域執行環境的東西，減少開發時因相同命名相衝的bug。

閉包 closure

  • JS的特性之一：即便這個變數所在的執行環境已經結束，jS引擎仍會確保，其它函式執行時可以找到它所需的變數。
