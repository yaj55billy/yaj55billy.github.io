Node.js 模組(內建)

Node.js codeNote


global

https://nodejs.org/dist/latest-v10.x/docs/api/globals.html#globals_global

  • Node global 跟瀏覽器的 window 不太相同
  • Node.js 比較嚴謹，每隻 JS 檔案就像一個模組，每個模組內的變數就是供這個模組使用，並不會汙染到全域的 global 。
  • 除非我們是用這樣的方式 >> global.a = 1

require、module exports 模組

現有兩隻檔案，如果我們要在 app.js 載入另一個 data.js，我們需要設定要求(require)跟輸出(module.exports)。

http 模組開啟 Web 伺服器 – createServer

如上程式碼部分，為開啟一個本地端的 web 伺服器。

  • http 是 Node 內建模組，所以可以直接做 require
  • req(request): 使用者透過瀏覽器輸入網址時，會有一個 request 的請求，這時 Node.js 會拿到 request 的資訊，例如從哪個 url 進來
  • resp (response): 回傳資料
  • Content-Type
    • text/plain >>文字格式
    • text/html >> html 格式
    • application/json >> json格式
  • listen 監聽並開啟 port，想像電腦每個軟體都佔有一個 port，8080為網頁開發常用，因為這個 port 較少被其他軟體佔據。
    • 21 >> FTP
    • 80 >> http
    • 3389 >> 遠端桌面

dirname、filename

  • __dirname: 應用程式的資料夾路徑
  • __filename: 應用程式的資料夾路徑跟檔案名稱
  • 設計路徑時會使用到

PATH

文件網址

  • 處理路徑相關模組 returns 路徑
  • 針對路徑抓出我們要的部分
  • basename: 抓檔名
  • extname: 抓副檔名
  • dirname: 抓檔案路徑
  • join: 路徑合併
  • parse: 返回一個陣列，裡面有 root、dir、base、ext、name 屬性
