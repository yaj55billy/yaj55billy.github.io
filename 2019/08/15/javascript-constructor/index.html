JavaScript 原型繼承、建構式

前言

最近因為有機會要用 JS 寫到小遊戲，而裡頭可能會有一些遊戲項目，例如:球、人物、障礙物、場景等等。在看別人的小遊戲程式碼時，發現滿多都會使用到物件來整理、原型建構子來做功能。故這邊會來整理下自己所學的。

一開始本來想先研究一下物件導向，但 Google 一輪後還是覺得霧傻傻，可能是自己接觸的程式語言不多(菜鳥)，過程會查到一些用詞，像是: 類別、多型、繼承，或者是會看到有人說 JS 不是物件導向的語言，他只是透過原型的概念來模擬。所以菜鳥的我就決定暫時不研究物件導向，直接來熟悉 JS 的原型概念，以及它實際上的運用等等。

這次主要會以 online game 的角色相關來做範例，透過以前打遊戲的經驗，這樣整理起來自己是比較好吸收些 XD

建構子函示 (建構式)

JavaScript 使用稱為建構子函式（constructor function）的特殊函式，定義物件與功能。開發者常常會不知道到底需建立多少物件，這時建構子可讓你高效率建立所需物件，並依需要為其添加資料與函式。
在新的物件實例透過建構子函式產生後，其核心將透過一種稱為原型鏈（Prototype chain，由原型定義，可參閱 Object prototypes）的參照鏈連在一起。

這段是在 MDN 物件導向看到的，那這段是什麼意思呢? 我們可以先往下看看範例，後面會再說明；另外這邊的說明會比較粗淺些，如果是要比較完善去了解，可以到 卡斯伯-JS 建構式。

心得

之前在看 JS 的奇怪部份時，就有先接觸到這個概念，但那時一直在思考要怎麼發揮(實作)，剛好借著這機會再多了解；雖然後面真的寫小遊戲時，也不確定是否能用到 (後來查資料時，發現有一些遊戲引擎可以使用)。

依照目前開發的案子來說，可能暫時也比較少機會用到 prototype 這個概念；不過我們在滿多套件的原始碼跟起手式都能看到建構函式的蹤影 XD 有一種越來越”稍微”看得懂別人寫這些套件的方式。
