Vue-Note

2019-05-29

Vue 的前置了解:

  • Vue 的開發版本有較多的錯誤提示、能使用 Chrome 的 Devtools (類似 js 的 console.log() )
  • Vue Devtools 連結

Vue 的起手式:

  • Vue 的應用程式，一次只能綁定一個 HTML 的元素，故建議綁定 #id 較好
  • 不能做巢狀，只會讀取到最外層有綁定的Vue應用程式

Vue 與 Mvvm:

  • Vue是受到 MVVM的啟發，不完全是MVVM
  • Mvvm
  • M Model 操控的內容 (資料)
  • V View 外觀視覺
  • VM ViewModel 自動綁定的部分
  • Mvvm定義
  • 透過 v-model 來操控資料 View (mvvm的概念)

v-text、v-html的輸出方式:

  • 除了雙括號的方式也可以使用 v-text、v-html
  • 差別是一個對文字 一個可對 html的標籤

v-bind 動態新增 HTML 的屬性

v-for、v-if

  • 就像我們使用 js 的 for迴圈跟 if判斷

v-on 操作頁面的行為(操作 data )

  • 就像我們用 jQ 的 onClick一樣

修飾符

  • 在 HTML裡面，可以用 v-on 來監聽一些常用的事件 (ex: event.preventDefault()……)
  • 也有一些按鍵的修飾符 (enter、esc、tab)
  • Vue修飾浮
  • Question: 為何 Vue的 methods內 this會指向data這個物件?

v-bind:class練習

  • 像是使用 jQuery toggleClass active一樣

computed

  • 是在監控資料更動後，重新運算結果呈現於畫面上，一般來說不會修改資料，只會回傳用於畫面呈現的資料
  • methods 比較用於互動的時候
  • 在 computed 裡面宣告的屬性都是 function(){}
  • 必須要找 data內的值做運算，如果data內沒有這個值，computed是沒有作用的

Vue基礎整理

    1. 創建跟 Vue的聯繫 >> HTML(畫面) 跟 Vue(資料) 的關係
    • var app = new Vue({ … })
    1. 將資料(Vue)呈現於畫面(HTML)上
    • 單一資料的狀況 >> 雙括號、v-text、v-html
    • 一筆資料的話(陣列) >> v-for迴圈渲染 、v-if的判斷
    1. 操作資料，將其帶至畫面上或者影響屬性(class、src…)
    • HTML input >> v-model 綁定資料的一個變數，在我們輸入時就能更改這個data值 (雙向綁定)
    • v-on:click (或其他鍵盤操作)，配合 Vue 的 methods (例如:按鈕點擊會做一些事情)
    • v-bind 操作屬性，將資料帶到 HTML的屬性
    • Vue 的 computed，可直接將運算結果 return到畫面上
