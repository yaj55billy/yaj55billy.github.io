ES6 let const 筆記

let 跟 const 基本上跟 var 一樣都是拿來宣告變數的，不過 ES6 才有的 let 跟 const 修正了原先 var 宣告時會遇到的問題。

使用 var 宣告會遇到的狀況:

  1. 多行程式碼重覆宣告

假設歷經很多程式碼，我們忘記前面是否有使用到 myName 變數時，這時後者的 myName 變數就會去蓋前者。

  1. 汙染全域變數

當我們迴圈使用 var 或者判斷式使用 var 時，都可以在全域讀取到這個值(汙染到全域變數)。

使用 var 不汙染全域變數的方式: 立即函式限制作用域

使用 ES6 的 let(可更改)、const(常數不可更改) 宣告:

  1. 多行程式碼重覆宣告

假設歷經很多程式碼，我們忘記前面是否有使用到 myName 變數時，這時使用 let 宣告時就會在 devTool 看到 Identifier ‘myName’ has already been declared (已經被宣告)

  1. 不汙染全域變數:

如果我們使用 let 來宣告變數時，i 這個變數就只會在這個迴圈區塊{ }作用，而不會汙染到全域。

  1. const 宣告的變數不能更改:

使用 const 時，則會在 devtool 看到 Assignment to constant variable.
